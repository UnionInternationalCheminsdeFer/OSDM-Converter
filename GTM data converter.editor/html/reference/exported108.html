Format considerations

    The IRS 10108.1 is not clear on code tables to be used and in some parts contardictory as the ASC-II character set is required but at the same time some elements sould contain special characters. The files created will apply charactersets for byte size encoding. Usually these will be the default character sets of the country. Except for data elements where special characters are indicated in 10108.1 all elements will be transliterated to ASC-II.

Unsupported 108 features

    The 108 files created from OSDM data should provide the basic features of the specification 10108.1. Some features on 10108.1 are dealing with data management can not be supported as they are not needed and implemented in OSDM.

    The OSDM data delivery policy is always to deliver the entire data set and not to indicate data changes. Therefore all flags indicating deletions or changes are not used. The 108 data will only include valid data and the flags are set to new

TCV file

  • indications on amendments for flags are not provided

TCVP file

  • fares are provided as they are defined in OSDM. Additional rounding as described in 10108.1 is not applied.
  • discount is not supported
  • number of children is not supported, only adult fares
  • replaced fare table is not supported
  • version number is always 1

TCVG file

Station data are exchanged via MERITS. The OSDM data include only a limited set on station names as these are currently not part of MERITS data.

  • old railway code is not supported
  • zone code is not supported
  • pictogram codes are not supported
  • fonts are not supported
  • designation for third party is not supported
  • accounting station code is not supported
  • geo-coordinates are supported for real stations only and contain the coordinates of MERITS. The standard WGS84 system is used as in MERITS, not the outdated Gauss-Krüger system.
  • version number is always 1
  • names of virtual border stations are constructed from real station names. There is no guarantee that border station names of two carriers are identical, the name must not be used to validate a connection of two series.

TCVS file

  • connecting codes are not supported
  • route number is provided as unique number but does not indicate a preference
  • product code is not supported
  • product offer code is not supported
  • indication of usual route is not supported
  • fare calculation will always be route based
  • ferry link is not supported
  • replaced series are not supported
  • abridging code will always be 0 as only necessary via stations are included in OSDM
  • version number is always 1

Price Table

  • distance based price table will not be used
  • no support for flags
  • the data version is always 1

TCVM file

  • the data version is always 1

TCVC file

The data for the TCVC file are not part of OSDM but taken from the RICS code table of UIC.

  • Address is not supported
  • RU managing the system is not supported
  • the data version is always 1

TCVL file

The 10108.1 specification contains an error in the format definition as element 3 is only one byte and not 5 bytes as indicated. The format created by the converter follows the format used by the railways (length of element 3 is 1 byte).

  • the data version is always 1
